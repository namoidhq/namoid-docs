title: Sign-in methods
description: Configure the sign-in methods currently available for a NamoID environment.
---

Hosted Auth displays the sign-in methods enabled for the selected environment.
Use the environment's **Authentication** and **Identity providers** pages in
the Console. Only methods shown as available there should be treated as
supported for that environment.

## Available categories

| Method | User experience |
|---|---|
| Email OTP | Enters an email and receives a one-time code |
| Password | Signs in with an email and password |
| Passkey | Uses a device passkey when enabled |
| External provider | Continues with a provider configured for the environment |

Provider availability and setup requirements vary. The Console is the source
of truth for credentials, callback URLs, and whether a provider can be enabled.

Do not build separate credential forms against private Hosted Auth endpoints.
Redirect users through the supported [Hosted Auth flow](/guides/hosted-login).
