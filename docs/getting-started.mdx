title: Getting started
description: Create a NamoID project and add a working Hosted Auth sign-in flow.

This walkthrough creates a Hosted Auth integration. You need a NamoID account
and a callback URL in your application.

## 1. Create a tenant and project

Sign in at [console.namoid.in](https://console.namoid.in/login). Create a
tenant, then create a project. Each project has isolated Test and Live
environments with separate users, credentials, configuration, and audit data.

Use **Test** while integrating. Its hosted environment URL has this form:

```text
https://<slug>.sandbox.namoid.in
```

Live uses `https://<slug>.id.namoid.in`.

## 2. Configure Hosted Auth

In the selected environment:

1. Enable the sign-in methods you want under **Authentication**.
2. Add your exact callback URL and browser origin under **Hosted Auth**.
3. Create an **Auth publishable** key for a browser or mobile client, or an
   **Auth secret** key for a server-rendered or backend-for-frontend client.

Publishable keys are safe to embed in a browser, but are restricted to the
configured origins. Secret keys must remain on the server.

## 3. Install an SDK

For a browser application:

```bash
pnpm add @namoidhq/js
```

Create a client with the Test publishable key:

```ts
import { createNamoIDClient } from "@namoidhq/js";

export const namoid = createNamoIDClient({
  publishableKey: "namoid_auth_pk_test_…",
});
```

The client resolves the environment's Hosted Auth URL from its public config.
You may pass `hostedAuthBaseUrl` explicitly when needed.

## 4. Start sign-in

Public clients must create a state and S256 PKCE transaction before redirecting:

```ts
const transaction = await namoid.hostedAuth.createPublicTransaction();
sessionStorage.setItem("namoid_transaction", JSON.stringify(transaction));

await namoid.hostedAuth.redirect({
  mode: "sign_in",
  returnTo: "https://app.example.com/auth/callback",
  state: transaction.state,
  completionMode: "public",
  codeChallenge: transaction.codeChallenge,
  codeChallengeMethod: transaction.codeChallengeMethod,
});
```

Use `mode: "sign_up"` or `mode: "waitlist"` for those flows.

## 5. Complete the callback

At the callback, compare the returned `state` with the saved transaction before
exchanging the one-time code:

```ts
const params = new URLSearchParams(window.location.search);
const code = params.get("code");
const state = params.get("state");
const transaction = JSON.parse(
  sessionStorage.getItem("namoid_transaction") ?? "null",
);

if (!code || !transaction || state !== transaction.state) {
  throw new Error("Invalid Hosted Auth callback");
}

sessionStorage.removeItem("namoid_transaction");
const tokens = await namoid.hostedAuth.exchangeCode({
  code,
  codeVerifier: transaction.codeVerifier,
});
```

Treat returned tokens as credentials. A production browser application should
prefer a backend-for-frontend that stores tokens in a server-side session. The
[Next.js SDK](/sdks/nextjs) handles the Hosted Auth transaction on the server.

## 6. Verify the integration

After signing in, confirm that the user appears under the selected
environment's **Users** view. Test and Live users are intentionally separate.
