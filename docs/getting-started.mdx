---
title: Getting started
description: Add a NamoID Hosted Auth sign-in flow to a browser application.
---

This guide uses the published `@namoidhq/js` package and a Test environment.

## 1. Create a project

Sign in at [console.namoid.in](https://console.namoid.in/login), create a
tenant, and create a project. Select the Test environment while integrating.

## 2. Configure Hosted Auth

In the selected environment:

1. Enable one or more sign-in methods shown as available.
2. Add your exact callback URL and browser origin under **Hosted Auth**.
3. Create an Auth publishable key.

Use the values shown in the Console. Do not use a secret key in browser code.

## 3. Install the JavaScript SDK

```bash
pnpm add @namoidhq/js
```

```ts
import { createNamoIDClient } from "@namoidhq/js";

export const namoid = createNamoIDClient({
  publishableKey: "namoid_auth_pk_test_…",
});
```

## 4. Start sign-in

Create a state and PKCE transaction before redirecting:

```ts
const transaction = await namoid.hostedAuth.createPublicTransaction();
sessionStorage.setItem("namoid_transaction", JSON.stringify(transaction));

await namoid.hostedAuth.redirect({
  mode: "sign_in",
  returnTo: "https://app.example.com/auth/callback",
  state: transaction.state,
  completionMode: "public",
  codeChallenge: transaction.codeChallenge,
  codeChallengeMethod: transaction.codeChallengeMethod,
});
```

## 5. Complete the callback

Validate the returned state before exchanging the one-time code:

```ts
const params = new URLSearchParams(window.location.search);
const code = params.get("code");
const returnedState = params.get("state");
const raw = sessionStorage.getItem("namoid_transaction");
const transaction = raw ? JSON.parse(raw) : null;

if (!code || !transaction || returnedState !== transaction.state) {
  throw new Error("Invalid Hosted Auth callback");
}

sessionStorage.removeItem("namoid_transaction");
const tokens = await namoid.hostedAuth.exchangeCode({
  code,
  codeVerifier: transaction.codeVerifier,
});
```

Treat returned tokens as credentials. Prefer a server-side application session
for production web applications.

## 6. Verify the integration

Complete a Test sign-in and confirm that the user appears in the selected
environment's **Users** view.
