NamoID provides hosted sign-in and sign-up pages. Your application redirects a
user to NamoID and receives a one-time code after authentication. Use a NamoID
SDK to exchange that code for a NamoID session.

## Supported integration

- Hosted authentication pages for Test and Live environments.
- Sign-in methods exposed as available in the NamoID Console.
- Separate Test and Live users, keys, and settings.
- Published JavaScript, React, and Next.js SDKs for Hosted Auth.

## Resource hierarchy

```text
Tenant
└── Project
    ├── Test environment  https://<slug>.sandbox.namoid.in
    └── Live environment  https://<slug>.id.namoid.in
        ├── Auth keys
        └── End users
```
