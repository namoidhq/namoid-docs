title: NamoID Auth flow

---

NamoID Auth is the hosted authentication UI NamoID renders for your users. Your
application redirects to it instead of collecting credentials itself.

## Environment hosts

```text
Test: https://<slug>.sandbox.namoid.in
Live: https://<slug>.id.namoid.in
```

Use the NamoID Auth base URL shown in the Console or returned by public auth
configuration. Do not construct it from the project name in application code.

## Supported flow

1. Redirect to `/sign-in`, `/sign-up`, or `/waitlist` with `return_to`, `state`,
   and a completion mode.
2. Public clients include an S256 PKCE challenge.
3. NamoID authenticates the user and redirects to `return_to` with a one-time
   `code` and the original `state`.
4. Verify `state`, then exchange the code at `POST /v1/auth/hosted/exchange`.

Use an Auth publishable key for a public browser client or an Auth secret key
for a confidential server integration. Keep secret keys on the server.

The [JavaScript](/sdks/javascript), [React](/sdks/react), and
[Next.js](/sdks/nextjs) SDKs implement this flow.

## Sign-out

Clear your application's own session when signing out. The Next.js SDK can also
revoke the NamoID session and redirect through the environment's `/sign-out`
page.
