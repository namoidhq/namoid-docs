Hosted Auth is the authentication UI NamoID renders for your end users. Your
application redirects to it instead of collecting credentials itself.

## Environment hosts

```text
Test: https://<slug>.sandbox.namoid.in
Live: https://<slug>.id.namoid.in
```

The same environment host serves sign-in UI and account pages. Test and Live
remain fully isolated.

## Native Hosted Auth

This is the recommended path for new NamoID SDK integrations.

1. Redirect to `/sign-in`, `/sign-up`, or `/waitlist` with `return_to`, `state`,
   and a completion mode.
2. For a public client, include an S256 PKCE challenge.
3. NamoID authenticates the user and redirects to `return_to` with a one-time
   `code` and the original `state`.
4. Verify `state`, then exchange the code at `POST /v1/auth/hosted/exchange`.

Use an Auth publishable key for public clients or an Auth secret key for a
confidential server integration. One-time codes are short lived and cannot be
reused.

The [JavaScript](/sdks/javascript), [React](/sdks/react), and
[Next.js](/sdks/nextjs) SDKs implement this flow.

## Configuration and security

The selected environment controls enabled sign-in methods, MFA policy, access
mode, waitlist state, consent behavior, and branding. Passwords, OTPs, passkeys,
and external identity-provider tokens remain within the hosted flow rather than
passing through your application.

## Rate limits

Verification attempts are rate limited per target and source to reduce
credential stuffing, OTP abuse, and automated guessing. OTP sends also respect
the selected environment's monthly spend cap.

## Sign-out

Hosted Auth uses `/sign-out`; the Next.js SDK can revoke the native session and
redirect there. Your application must also clear its own local session.

## End-user account pages

Authenticated users can open `/account` on their environment host to manage
profile information, sessions, connected identities, passkeys, and data
requests. Available controls follow the environment's configuration.

## Custom domains

A verified custom domain can replace the default environment hostname. Use the
Hosted Auth base URL shown by the environment after verification; do not
construct endpoint URLs from the project name.
